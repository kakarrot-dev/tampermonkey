# 功能规格：X 阅读增强（二期）

## 背景

一期已隐藏右栏并将主栏加到约 920px，但宽屏仍大量右白、阅读密度偏疏，右下 Grok/消息浮钮落在空白区更显干扰。需要在纯 CSS 前提下做布局吃满、加压密度、减干扰。

一期规格：`docs/superpowers/specs/2026-07-20-x-ux-css-spec.md`。

## 目标

1. 布局方案 A：藏右栏后主栏 `flex:1` 吃满空位，上限约 1200px；布局根同步加宽。
2. 加压阅读密度：正文行高、帖间距、信息流媒体最大高度。
3. 隐藏右下 Grok、消息两个浮钮。
4. 保留左导航；新旧布局壳选择器兼容。

## 非目标

- 左导航图标化 / 隐藏
- 顶部分类 Tab 折叠
- 配置面板、主题换色、广告识别
- JS DOM 重排、自动互动、私有接口

## 用户场景

桌面浏览器用 Tampermonkey 打开 x.com，希望主栏真正占满右栏空位、一屏更多文字、少浮层打扰。

## 需求边界

包含：在现有 `GM_addStyle` 脚本上扩展 CSS。  
不包含：存储、开关 UI、扩大 `@match` / 其它 GM API。

## 匹配范围

```text
@match https://x.com/*
@match https://twitter.com/*
```

## 交互流程

更新脚本后硬刷新即生效；SPA 跳转后样式保持。无额外控件。

## 数据与状态

无存储。

## 权限与安全

```text
@grant GM_addStyle
```

不碰登录态与隐私数据。

## 异常情况

- 未登录无三栏：样式无目标可作用，属预期。
- 浮钮 DOM 改版：按 `docs/selectors.md` 重测更新选择器。
- 详情页大图若被 `max-height` 误伤过重：收窄媒体选择器至时间线语境。

## 验收标准

- 1280 / 1440 / 1920：右栏隐藏、左导航在、主栏明显宽于一期 920，右白明显减少
- 正文更密、大图不轻易占满一屏、媒体圆角仍在
- 右下 Grok / 消息浮钮不可见
- 首页 / 个人主页 / 详情 SPA 跳转后样式仍在
- 控制台无脚本报错
